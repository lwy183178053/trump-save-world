# 全局数据与配表规范

## 1. 运行时数据中心

运行时只保留两层：

- `GameDataCenter`：管理会变化的数值状态
- `GameEventBus`：管理离散事件

### 1.1 数据中心规则

- 所有运行时数值只通过 `GameDataCenter` 改
- 改值时自动广播 `GameDataChangedEvent`
- 支持晚绑定回放当前值
- 支持批量修改，避免一次效果触发多次刷新

### 1.2 已有核心键

- `A_1`：支持率
- `D_1`：威慑值
- `H_1`：心率

建议后续继续按这个 ID 风格扩展。

### 1.3 事件源

建议统一标记来源：

- `PlayerAction`
- `UI`
- `Gift`
- `Proposal`
- `Event`
- `Restore`
- `Replay`

这样 UI、音效、数值逻辑都能知道这次变化是怎么来的。

## 2. 配表总原则

### 2.1 ID 规则

- 使用 `类型_序号`
- 类型保持短且稳定，例如 `A_1`、`D_1`、`H_1`、`G_1`、`AS_1`
- 一个 ID 只表达一个独立对象，不要复用

### 2.2 表的职责

- `总览表`：目录，不放玩法值
- `变量表`：放会被运行时修改的数值
- `效果表`：放一次触发会改哪些值
- `事件表`：放触发规则、概率、冷却、持续
- `文案表`：放文本、图片编号、音效引用

## 3. 推荐 sheet 结构

### 3.1 总览表

字段：

| 字段 | 说明 |
| --- | --- |
| ID | 条目标识 |
| 类型 | 表类型 |
| 表名 | 对应 sheet 名 |
| 说明 | 这张表的用途 |
| 引用键 | 关联到的主 ID |
| 备注 | 可选说明 |

用途：

- 给策划查表
- 给导表脚本做索引

### 3.2 变量表

一行一个运行时变量。

字段：

| 字段 | 说明 |
| --- | --- |
| ID | 变量 ID |
| 变量名 | 中文名 |
| 类型 | `int / float / bool` |
| 初始值 | 开局值 |
| 最小值 | 下限 |
| 最大值 | 上限 |
| 步长 | 单次变化量 |
| 单位 | `% / 秒 / 点` |
| 自动变化公式 | 定时变化规则 |
| 变化来源 | 谁会改它 |
| 说明 | 设计备注 |

### 3.3 效果表

一行一个效果，不区分礼物、推特、按钮，统一视作“效果”。

字段：

| 字段 | 说明 |
| --- | --- |
| 效果ID | 效果标识 |
| 目标变量ID | 改哪个变量 |
| 运算类型 | `Add / Set / Mul / Clamp` |
| 数值 | 变化量 |
| 条件 | 触发前置条件 |
| 持续时间 | 持续型效果时长 |
| 冷却时间 | 再次触发间隔 |
| 优先级 | 多效果冲突时排序 |
| 说明 | 备注 |

### 3.4 事件表

一行一个事件规则。

字段：

| 字段 | 说明 |
| --- | --- |
| 事件ID | 事件标识 |
| 事件类型 | 干扰 / 暗杀 / 礼物 / 提案 / 系统 |
| 触发条件 | 触发公式或条件描述 |
| 触发概率 | 百分比 |
| 触发间隔 | 多少秒判定一次 |
| 持续时间 | 持续型事件用 |
| 效果ID | 命中的效果 |
| 文案ID | UI 文本引用 |
| 说明 | 备注 |

### 3.5 文案表

一行一个文本资源。

字段：

| 字段 | 说明 |
| --- | --- |
| 文案ID | 文本标识 |
| 类型 | 推特 / 对话 / 提示 / 结算 |
| 内容 | 文本本体 |
| 图片编号 | 配图 ID |
| 音效ID | 可选 |
| 说明 | 备注 |

## 4. 结合当前 Excel 的建议

你现在那份表可以按下面方向收敛：

- `支持率值配置表` -> 变量表
- `威慑值配置表` -> 变量表
- `心率值配置表` -> 变量表
- `礼物数值配置表` -> 效果表
- `推特数值配置表` -> 效果表
- `事件数值配置表` -> 事件表
- `推特文本` / `对话文本` -> 文案表

## 5. 代码侧建议

UI 绑定示例思路：

- `ObserveValue("A_1", ...)` 绑定支持率
- `ObserveValue("D_1", ...)` 绑定威慑值
- `ObserveAll(...)` 做调试面板或通用日志

离散事件示例思路：

- `Publish(GameEventNames.ButtonPressed)`
- `Publish(GameEventNames.GiftReceived, giftId)`
- `Publish(GameEventNames.GameLost)`

## 6. 约定

- 数值变化走 `GameDataCenter`
- 离散行为走 `GameEventBus`
- UI 不直接轮询数值
- 未来导表再把 Excel 转成 JSON 或 ScriptableObject
